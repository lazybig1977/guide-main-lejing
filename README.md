# guide-main-lejing

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可以是工具、说明页、导航页或其他静态内容，不针对特定域名或网站。

## 项目简介

这是一个通用的 HTML 页面归档仓库，主要目的是集中存放和发布静态 HTML 文件。仓库中的每个页面都是独立、自包含的，不依赖外部资源（除非在页面内显式引用）。

适合存放：
- 小型工具页面
- 说明文档页
- 个人导航页
- 其他需要直接通过浏览器打开的 HTML 内容

## 目录说明

仓库根目录下直接存放 HTML 文件。每个文件建议命名清晰，便于识别。

```
/
├── index.html        # 可选的主页
├── page-example.html # 示例页面
└── README.md         # 本文件
```

如页面较多，可按功能或类型创建子目录，例如：

```
/
├── tools/
├── docs/
└── pages/
```

## 页面归档说明

- 每个 HTML 页面应保持独立，尽量使用内联样式和脚本，减少外部依赖。
- 页面标题、描述等元信息建议在 HTML 的 `<head>` 中明确声明。
- 如果页面引用外部资源（如图片、字体、库），请确保资源链接长期有效或一并归档。

## 维护说明

- 欢迎提交 Pull Request 添加新的 HTML 页面，或改进现有页面。
- 请确保新增页面不包含恶意代码、跟踪脚本、商业推广或违反平台政策的内容。
- 本仓库主要用于归档和静态发布，不建议存放动态内容或需要后端支持的应用。
- 如页面涉及敏感或专有内容，请自行评估是否适合公开。
